Submission Loading UX

When the user clicks "$ roast_my_code", what happens while the AI is analyzing? (~5–15s)

A

Wait on home, then redirect

Button shows a spinner / "$ analyzing..." state. User stays on the home page until the full AI response is ready, then gets redirected to /roast/[id]. Simple, no intermediate page needed.

[home page]
$ analyzing... ← button state
↓ (10s later)
redirect → /roast/abc123
B

Immediate redirect to a loading results page

Click submits instantly, user is redirected right away to /roast/[id] which shows a skeleton/loading state. The page polls or revalidates until the roast is ready, then reveals the result.

[home page]
$ roast_my_code → redirect immediately
↓
/roast/abc123 ← skeleton screen
↓ polling / revalidate
results appear
C

Streaming results page

Redirect immediately to /roast/[id]. The AI response streams in — score, verdict, issues appear progressively as they're generated. Most dynamic, most complex to build.

[home page] → redirect immediately
↓
/roast/abc123
score: ... ← streams in
verdict: ... ← streams in
issues: [...] ← streams in
